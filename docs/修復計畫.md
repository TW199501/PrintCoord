### 核心缺失項目

- **TypeScript 設定仍指向舊 monorepo**[tsconfig.json](cci:7://file:///d:/app/table-template/tsconfig.json:0:0-0:0) 延伸 `../../../tsconfig.json` 並 include `../../../components` 等外部路徑，新的單獨專案會找不到這些檔案。@tsconfig.json#1-25
- **Path Alias 與 shadcn 元件來源遺失**例如 [TemplateManager](cci:1://file:///d:/app/table-template/TemplateManager.tsx:40:0-487:1) 仍從 `@/components/ui/*` 匯入，但目前的 [components/](cci:7://file:///d:/app/table-template/components:0:0-0:0) 目錄只有業務元件，沒有 `ui` 子資料夾或 `lib/utils.ts`。@TemplateManager.tsx#5-15同時 [tsconfig.json](cci:7://file:///d:/app/table-template/tsconfig.json:0:0-0:0) 未設定 `paths`，`@/...` 別名會編譯失敗。@tsconfig.json#3-25
- **Tailwind / PostCSS 組態不完整**

  - [package.json](cci:7://file:///d:/app/table-template/package.json:0:0-0:0) 缺少 `tailwindcss`, `postcss`, `autoprefixer` 等核心依賴，也沒有 `tailwindcss-animate` 對應版本。@package.json#33-54
  - 專案沒有 `postcss.config.js`，建制會無法處理 `@tailwind` 指令。
- **Next.js 依賴位置錯誤**`react`, `react-dom`, `next` 被放在 `peerDependencies`，但這裡是實際執行的 Next App，需要移到 `dependencies` 才能安裝。@package.json#28-42
- **shadcn 對應資源缺片**
  [tailwind.config.ts](cci:7://file:///d:/app/table-template/tailwind.config.ts:0:0-0:0) 已載入 `tailwindcss-animate`，但專案缺少 shadcn 產生的 UI 元件與 CSS token（如 `components/ui/button.tsx`, `lib/utils.ts` 等），僅保留舊專案 transpile 後的 [dist/](cci:7://file:///d:/app/table-template/dist:0:0-0:0)。@tailwind.config.ts#1-77

### 建議修復步驟

1. **重建 TypeScript 設定**

   - 將 `extends` 改成官方 `next/tsconfig.json` 或自建基底。
   - 調整 `include/exclude` 只涵蓋本 repo。
   - 加入 `compilerOptions.paths` (`"@/*": ["./*"]`) 以支援 `@/...`。
2. **補齊 Tailwind & PostCSS**

   - 安裝 `tailwindcss`, `postcss`, `autoprefixer`, `tailwindcss-animate`。
   - 生成 `postcss.config.js`、[tailwind.config.ts](cci:7://file:///d:/app/table-template/tailwind.config.ts:0:0-0:0)（已存在可續用）並在 [package.json](cci:7://file:///d:/app/table-template/package.json:0:0-0:0) 增加 `postcss` script（例如 `next build` 前執行）。
3. **調整依賴分類**

   - 將 `react`, `react-dom`, `next` 移到 `dependencies`。
   - 若打算發佈成套件，再視情況保留 peerDependency，但需同時列在 devDependencies 以本地開發。
4. **恢復 shadcn/ui 原始檔**

   - 從原專案或使用 `npx shadcn@latest add ...` 重新生成 `components/ui/*` 及 `lib/utils.ts`。
   - 依需要補回 `styles`/`fonts`/`config` 等 shadcn scaffolding。
5. **建立 Next.js 組態**

   - 新增 `next.config.mjs`（若有客製需求，例如 `images`, `transpilePackages` 等）。
   - 更新 [package.json](cci:7://file:///d:/app/table-template/package.json:0:0-0:0) scripts（`next dev`, `next build`, `next lint`）。

照以上步驟補齊，即可讓拆出後的專案具備完整的 Next.js + Tailwind + shadcn/ui 組態，再逐一測試頁面與服務模組。
